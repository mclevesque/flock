## 🗄️ Database — Neon Postgres

- Connection string is in `.env.local` as `DATABASE_URL`
- All queries use the `sql` tagged template literal from `@neondatabase/serverless`
- Schema migrations use `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` (safe, never destructive)
- Tables: `users`, `friendships`, `messages`, `group_messages`, `watch_rooms`, `watch_room_members`, `emulator_rooms`, `chess_games`, `quiz_challenges`, `quiz_games`, `poker_rooms`, `poker_players`, `voice_rooms`
- Always call `ensureXxxTables()` at the top of API routes that need a table to exist

## 🏗️ Stack

- **Framework**: Next.js 14 App Router (`app/` directory, not `pages/`)
- **Auth**: NextAuth.js — session via `getServerSession()` server-side, `useSession()` client-side

- **WebRTC**: P2P screen share in `/app/stremio/[id]/WatchRoom.tsx`

## 🎮 Feature Notes

### Watch Rooms (`/stremio`)
- `invite_only` boolean column on `watch_rooms` — if true, room is hidden from public lobby and friends list
- Rooms with no members auto-close after **20 minutes** via `closeVacantWatchRooms()` called before queries
- Host can toggle invite-only with 🔒/🌐 button (host-only UI)
- Friends in watch rooms visible in DM sidebar + VoiceWidget via `GET /api/friends/watch-rooms`

### Arena Rooms (`/emulator`)
- Rooms persist until host explicitly closes them (host gets "Close Room ✕", guests get "Leave ✕")
- Auto-close after **20 minutes vacant** (`game_started = false AND updated_at < NOW() - 20 min`)
- Heartbeat: `EmulatorClient` sends `action: "heartbeat"` every poll cycle to keep `updated_at` fresh
- State persisted in `sessionStorage` so refresh doesn't lose the room

### Quiz DMs
- Message format: `[quiz:challengeId]` — renders as `InlineQuizCard`
- Both challenger AND recipient poll every 4s for game status
- Game state recovered from `sessionStorage` key `iq_${challengeId}` on mount
- If `sessionStorage` is lost (page refresh), mount effect checks API for existing `gameId` and recovers
- Quiz questions generated by GROQ (LLaMA 3.3 70B), fallback to OpenTDB free API

### Error Boundaries
- `/app/stremio/error.tsx` — catches crashes on the lobby page
- `/app/stremio/[id]/error.tsx` — catches crashes in watch room
